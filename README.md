## Executive Summary

Upvista Community is a comprehensive social networking and professional collaboration platform designed to facilitate connections, content sharing, and real-time communication. The platform combines the engagement features of contemporary social networks with the professional networking capabilities of business-oriented platforms, creating a unified ecosystem for personal and professional interactions.

The system is built using a modern, scalable architecture that supports web, mobile, and desktop applications. It features enterprise-grade authentication, real-time messaging, content management, and social interaction capabilities, all designed to provide users with a seamless, secure, and performant experience.

**Strategic Position:** Upvista Community serves as a complete social platform solution, enabling users to create profiles, share content, connect with peers, engage in real-time messaging, discover communities, and participate in professional networking activities.

---

## Problem Statement

Modern social networking platforms face several critical challenges that limit user experience and platform effectiveness:

**Fragmentation of Professional and Personal Networks**
Traditional social platforms separate professional and personal networking, forcing users to maintain multiple accounts across different services. This fragmentation creates barriers to meaningful connections and content discovery.

**Inadequate Real-Time Communication**
Many platforms lack sophisticated real-time communication features, resulting in delayed message delivery, poor notification systems, and limited engagement opportunities. Users require instant, reliable communication channels that work seamlessly across devices.

**Limited Content Discovery and Curation**
Existing platforms struggle with effective content discovery, making it difficult for users to find relevant content, communities, and connections. Algorithm-based feeds often prioritize engagement over relevance, reducing user satisfaction.

**Insufficient Professional Tools**
Professional networking platforms lack the engagement features of social platforms, while social platforms lack professional networking capabilities. Users need a unified platform that combines both aspects effectively.

**Security and Privacy Concerns**
Many platforms lack enterprise-grade security features, proper authentication mechanisms, and privacy controls. Users require secure, transparent systems that protect their data and provide control over their information.

**Scalability and Performance Issues**
As platforms grow, they often encounter performance bottlenecks, slow response times, and poor mobile experiences. Users expect fast, responsive applications that work reliably at any scale.

**Solution:** Upvista Community addresses these challenges by providing a unified platform with robust real-time communication, sophisticated content discovery, professional networking tools, enterprise-grade security, and scalable architecture that delivers exceptional performance across all devices and use cases.

### Component Interaction Flow

**Authentication Flow:**
1. Client sends login request to Backend API
2. Backend validates credentials against PostgreSQL
3. Backend generates JWT token
4. Backend stores session in Redis
5. Backend returns token to client
6. Client stores token and includes in subsequent requests

**Real-Time Messaging Flow:**
1. Client establishes WebSocket connection to Backend
2. Backend authenticates WebSocket connection
3. Client sends message via WebSocket
4. Backend stores message in PostgreSQL
5. Backend caches message in Redis
6. Backend broadcasts message to recipient via WebSocket
7. Recipient receives real-time message update

**Content Creation Flow:**
1. Client uploads media to Supabase Storage
2. Client creates post via REST API
3. Backend processes and validates post
4. Backend stores post in PostgreSQL
5. Backend updates Redis cache
6. Backend triggers background jobs (hashtag extraction, notifications)
7. Backend broadcasts post via WebSocket to followers
8. Followers receive real-time feed update

### Response Format

**Success Response:**
```json
{
  "success": true,
  "message": "Operation completed successfully",
  "data": { ... }
}
```

**Error Response:**
```json
{
  "success": false,
  "error": "Error message",
  "code": "ERROR_CODE"
}
```

### Rate Limiting
- Authentication endpoints: 5 requests per minute per IP
- Registration endpoints: 3 requests per minute per IP
- Password reset: 3 requests per minute per IP
- General API: 200 requests per minute per authenticated user

### WebSocket API

**Connection URL:** `ws://localhost:8081/ws` (development) or `wss://api.upvista.com/ws` (production)

**Message Types:**
- `message.new` - New message received
- `message.typing` - Typing indicator
- `post.liked` - Post liked notification
- `post.commented` - Post commented notification
- `user.followed` - User followed notification
- `notification.new` - New notification

**Example WebSocket Message:**
```json
{
  "type": "message.new",
  "payload": {
    "id": "uuid",
    "conversation_id": "uuid",
    "sender_id": "uuid",
    "content": "Message content",
    "created_at": "2025-11-01T12:00:00Z"
  }
}
```

## Installation and Setup

### Prerequisites

**Required Software:**
- Node.js 20.x or higher
- Go 1.19 or higher
- Flutter 3.3.0 or higher (for mobile development)
- PostgreSQL 15 or higher (managed via Supabase)
- Redis 7.0 or higher (for caching)
- Git (version control)

**Required Accounts:**
- Supabase account (free tier available)
- SMTP email service (Gmail, SendGrid, or AWS SES)
- OAuth credentials (Google, GitHub, LinkedIn - optional)

### Database Setup

**1. Create Supabase Project**
- Sign up at [supabase.com](https://supabase.com)
- Create a new project
- Note your project URL, anon key, and service role key

**2. Run Database Migrations**
- Open Supabase SQL Editor
- Execute migration scripts in order:
  ```bash
  # Run initial migration
  scripts/migrate.sql
  
  # Run additional migrations as needed
  scripts/posts_system_migration.sql
  scripts/messaging_migration.sql
  scripts/notifications_migration.sql
  ```

**3. Configure Row Level Security**
- RLS policies are included in migration scripts
- Verify policies are active in Supabase Dashboard

**4. Set Up Storage Buckets**
- Create storage bucket for profile pictures
- Create storage bucket for post media
- Configure bucket policies for public/private access

### Backend Setup

**1. Navigate to Backend Directory**
```bash
cd backend
```

**2. Install Dependencies**
```bash
go mod tidy
```

**3. Configure Environment Variables**
Create a `.env` file in the `backend/` directory:
```env
# Server Configuration
PORT=8081
GIN_MODE=debug
CORS_ALLOWED_ORIGINS=http://localhost:3000,http://localhost:3001

# Database Configuration (Supabase)
SUPABASE_URL=https://your-project.supabase.co
SUPABASE_ANON_KEY=your-anon-key
SUPABASE_SERVICE_ROLE_KEY=your-service-role-key

# JWT Configuration
JWT_SECRET=your-secret-key-min-32-chars
JWT_EXPIRY=720h
REFRESH_TOKEN_EXPIRY=720h

# Email Configuration (SMTP)
SMTP_HOST=smtp.gmail.com
SMTP_PORT=587
SMTP_USERNAME=your-email@gmail.com
SMTP_PASSWORD=your-app-password
SMTP_FROM_NAME=Upvista Community
SMTP_FROM_EMAIL=your-email@gmail.com
FRONTEND_URL=http://localhost:3000

# Storage Configuration (Supabase)
BUCKET_NAME=profile-pictures
MAX_FILE_SIZE=5242880
ALLOWED_FILE_TYPES=image/jpeg,image/png,image/gif,image/webp

# Redis Configuration (Optional, for caching)
REDIS_HOST=localhost
REDIS_PORT=6379
REDIS_PASSWORD=
REDIS_DB=0

# OAuth Configuration (Optional)
GOOGLE_CLIENT_ID=your-google-client-id
GOOGLE_CLIENT_SECRET=your-google-client-secret
GITHUB_CLIENT_ID=your-github-client-id
GITHUB_CLIENT_SECRET=your-github-client-secret
LINKEDIN_CLIENT_ID=your-linkedin-client-id
LINKEDIN_CLIENT_SECRET=your-linkedin-client-secret
```

**4. Start Backend Server**
```bash
# Development mode
go run main.go

# Production build
go build -o upvista-backend main.go
./upvista-backend
```

**5. Verify Backend is Running**
```bash
curl http://localhost:8081/health
```

The backend server should respond with a health status.

### Frontend Web Setup

**1. Navigate to Frontend Directory**
```bash
cd frontend-web
```

**2. Install Dependencies**
```bash
npm install
```

**3. Configure Environment Variables**
Create a `.env.local` file in the `frontend-web/` directory:
```env
NEXT_PUBLIC_API_BASE_URL=http://localhost:8081/api/v1
NEXT_PUBLIC_WS_URL=ws://localhost:8081/ws
```

**4. Start Development Server**
```bash
npm run dev
```

**5. Access Web Application**
Open `http://localhost:3000` in your browser.

**6. Build for Production**
```bash
npm run build
npm run start
```

### Mobile Application Setup

**1. Navigate to Mobile Directory**
```bash
cd mobile-app
```

**2. Install Dependencies**
```bash
flutter pub get
```

**3. Configure Application**
- Update `lib/core/config/app_config.dart` with your API base URL
- Configure Android and iOS settings as needed

**4. Run on Device or Emulator**
```bash

flutter devices

# Run on specific device
flutter run -d <device-id>

# Build for Android

# Build for iOS
flutter build ios
```

**5. Run Code Generation (if needed)**
```bash
flutter pub run build_runner build --delete-conflicting-outputs
```

### Redis Setup (Optional)

```

**3. Verify Redis is Running**

```

Should respond with `PONG`.

### Complete System Verification

**1. Verify All Services are Running**
- Backend API: `http://localhost:8081/health`
- Frontend Web: `http://localhost:3000`
- Redis: `redis-cli ping` (if using Redis)
- Database: Verify connection in Supabase Dashboard

**2. Test Authentication Flow**
- Register a new user via frontend
- Verify email with code
- Login with credentials
- Verify JWT token is received

**3. Test Real-Time Features**
- Open messaging interface
- Verify WebSocket connection is established
- Send a test message
- Verify real-time delivery

**4. Test Content Creation**
- Create a text post
- Upload an image
- Verify post appears in feed

### Troubleshooting

**Common Issues:**

**Backend won't start:**
- Verify Go version: `go version` (should be 1.19+)
- Check environment variables are set correctly
- Verify database connection string is valid
- Check port 8081 is not in use

**Frontend build fails:**
- Verify Node.js version: `node --version` (should be 20.x+)
- Delete `node_modules` and `package-lock.json`, then run `npm install` again
- Check TypeScript errors: `npm run lint`

**Mobile app won't run:**
- Verify Flutter installation: `flutter doctor`
- Check Android/iOS SDKs are configured
- Verify API base URL in app config

**Database connection fails:**
- Verify Supabase credentials are correct
- Check network connectivity
- Verify database is active in Supabase Dashboard

**WebSocket connection fails:**
- Verify backend WebSocket server is running
- Check WebSocket URL in frontend config
- Verify CORS settings allow WebSocket connections
